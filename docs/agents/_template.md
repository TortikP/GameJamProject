# <Имя агента>

**Тэг:** `@<имя>`
**Статус:** прототип / стабильный / устаревший

## Роль

Одно предложение, описывающее кто этот агент. Не "что делает", а "кто он".

Пример: "Хранитель дизайн-памяти проекта. Скриб, не дизайнер."

## Активируется когда

- Явный тэг `@<имя>` в сообщении.
- Типы запросов: перечислить, какие задачи естественно ложатся на этого агента.
- Ключевые слова, по которым мозг может предложить роутинг к этому агенту: список.

## Делает

Конкретный список активных обязанностей. Не общие принципы — то, что агент **делает руками** в задачах.

- Действие 1.
- Действие 2.
- ...

## Не делает

То, что агент **отдаёт обратно мозгу или другому агенту**. Самая важная секция — она предотвращает расползание роли.

- Не делает X — это задача мозга / другого агента.
- Не делает Y.
- ...

## Тон

Одно-два предложения о манере общения. Сухой / развёрнутый, формальный / дружеский, краткий / подробный, и т.д.

## Хэнд-офф

Когда агент возвращает контроль мозгу. По умолчанию — после каждого ответа. Если есть особенности (например, "продолжает в роли, пока юзер не скажет иное") — описать.

## Чек-листы (опционально)

Если у агента есть повторяющиеся форматы выходов (запись в журнал, формат отчёта) — перечислить чек-листы качества здесь.

## История

Кто придумал этого агента, когда, и зачем. Помогает не переоткрывать решение через полгода.

**Обсуждали:** Имя1 (идея), Имя2.
