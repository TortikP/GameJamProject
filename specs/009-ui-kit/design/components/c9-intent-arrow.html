Spec — C9 Intent arrow
Purpose
Telegraph the enemy's next-tick action: where they will move, or that they will attack from where they stand. Pairs with C6 (which shows what they hit).
Anchor & sizing
World-space, parented to enemy. Offset (0, -28) px from sprite top. 24×24 box. Distance label sits at (0, +14) below the arrow box.
Inputs
kind:        enum { move, attack, idle }
direction:   enum { ne, e, se, sw, w, nw }   # only when kind == move
distance:    int                              # only when kind == move
team:        enum { player, enemy, neutral } # for color
visible:     bool                             # hide for off-screen / fog
Glyph table
kind = move:
  ne → ↗   e → →   se → ↘
  sw → ↙   w → ←   nw → ↖
kind = attack: ⊕
kind = idle:   —
Edge cases
• Multi-step path: arrow shows total distance + first-step direction. Path pips render only when this enemy is selected.
• Path blocked between plan and resolution: arrow stays; resolution may abort or re-route depending on AI rules. Mockup is the plan, not the actual outcome.
• Enemy stunned: show kind=idle with dur from stun status.
• Action invisible to player: arrow hidden if enemy is in fog-of-war.
Godot mapping
IntentArrow : Control                  # custom_minimum_size = (24, 24)
  PanelContainer                          # StyleBoxFlat: bg=#000@55%, border=team color
    Label glyph                           # 14pt mono bold, color = team
  Label distance                          # anchor bottom_center y=+14, 9pt mono
                                            # bg via theme override or PanelContainer
Direction is purely a glyph swap — no rotation transform needed (Unicode arrows cover all 6 directions). This matches the existing in-code widget and avoids rotation cost.
