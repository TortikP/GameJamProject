Spec — C22 / C23
Purpose
C22 is the entry point of the application — must be flawless and feel like the game's voice. C23 is the trust-fall: the player paused expecting their run to be safe; everything here is reversible except "abandon run".
Anchor & sizing
Centered modal, 360×auto. CanvasLayer above arena. C23 dims at 0.78 alpha; C22 backdrop is the title art at full opacity.
Inputs
// C22
save_present:  bool
last_run_meta: { wave, time, seed }?
build_string:  String

// C23
run_meta: { location, wave, turn, hp_pct, voices }
last_autosave_ago_s: int
seed: String
Outputs
signal new_run, signal continue_run, signal resume, signal abandon_confirmed, signal open_settings, signal quit
Keyboard contract
• ↑/↓ moves focus through enabled buttons only (skips disabled).
• Letter hotkeys (N, S, ?) jump focus and activate.
• Esc on C22 = quit; Esc on C23 = resume.
• Enter activates focused button.
Edge cases
• Save corrupted: Continue stays disabled with a tip "save unreadable" (C15-style).
• Pause during animation/transition: queue the open until current animation step finishes; never pause mid-FCT.
• Confirm-abandon variant: reuses the same panel — only the eyebrow/title/buttons swap. Pressing Esc returns to main pause buttons, not out of pause entirely.
