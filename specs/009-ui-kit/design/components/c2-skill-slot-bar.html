Spec — C2 Skill slot bar
Purpose
Bind player's 4 active skills to keyboard hotkeys; show ready/cooling/disabled state at a glance; serve as click target for slot-select.
Anchor & sizing
Bottom bar, horizontally centered. Total ~400×104 px (4 slots × 88 + 3 gaps × 12 + 8 padding × 2). Fixed size; does not grow.
Inputs (per slot)
hotkey:    string ("Q", "W", "E", "R")
skill:     Skill | null
cooldown:  int    # turns remaining, 0 = ready
disabled:  bool   # non-cooldown reason
active:    bool   # cast_mode == casting(this_index)
Outputs
signal slot_pressed(index: int)        # LMB or hotkey
signal slot_hovered(index: int, in: bool)
signal illegal_press(index: int)        # for SFX hook
Interactions
• LMB on slot → if ready: slot_pressed → cast_mode=casting(i); if cooling/disabled/empty: illegal_press + flash.
• Hotkey Q/W/E/R or 1/2/3/4 → same as LMB on corresponding slot.
• Same key/LMB on already-active slot → cancel cast (cast_mode=idle).
• Hover → slot_hovered(i, true); tooltip C14 fires after 400ms delay.
• ESC → cancel cast (handled at controller, not slot).
Edge cases
• Empty slot pressed: dim-flash, no signal beyond illegal_press.
• Cooldown decrements mid-frame: when cooldown reaches 0 in same frame as keypress, treat as ready (cooldown checked before flash).
• Player dead: all slots disabled, no flash on press (input swallowed upstream).
Active skill description
When any slot is active (cast_mode == casting(i)), a SkillDescription panel appears anchored 12 px above the bar, horizontally centered. Shows: skill name + hotkey + tags (range/aoe/cd/cost) + body text. Same data as C14 (hover tooltip), different anchor. Hidden when cast_mode == idle. Mirrors the slot's --ui-focus frame via a 3 px left accent border so the visual link to the active slot is obvious.
Focus trap (§7)
If any input element on screen has focus (SpinBox / LineEdit), Q/W/E/R go to that input. Slot-bar must request keyboard focus on mouse-enter into the arena viewport, or controller intercepts at _input() before _gui_input() (recommended).
Godot mapping
SkillBar : PanelContainer                # anchored bottom-center
  HBoxContainer                          # theme separation = 12
    SlotButton ×4 : Button                # custom_minimum_size = (88, 88)
                                            # focus_mode = NONE (controller drives selection)
      Label hotkey                         # anchor TL, font 12 mono, --ui-text-dim
      Label glyph                          # anchor center, font 28 mono
      Label name                           # anchor BL+BR, font 11, ellipsis
      Label cd_overlay                     # anchor full, font 32 mono, hidden if cooldown==0

                                            #   bg=--ui-bg-panel-2  border 1px --ui-border  radius 4
                                            # Active state: swap StyleBox with border=--ui-focus,
                                            #   inner inset of 1px (set border_width_inner = 1)
Disabled state uses Button.disabled = true; flash is a Tween on bg color (added later — mockup is the snapshot).
