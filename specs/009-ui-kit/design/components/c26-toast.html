C26 — Toast / notification

Non-blocking system messages anchored top-right. 4 severities, 3 visible at once, the rest collapse into a "+N more" pill. Combat logs use C27 instead — toasts are for system/meta events (autosave, modifier ready, controller disconnect).

Spec — C26 Toast
Purpose
Non-blocking system messages. Combat events (damage, heal, status applied) go through C11 floating combat text and C27 combat log — never through C26.
Anchor & sizing
Top-right, 12 px from viewport edge. Width 320 px (40% of viewport on small res). Toast height auto. Stack gap 8 px. Capped at 3 visible; older entries collapse into a "+N more" pill that expands a scrollable history.
Severities & defaults
INFO     8s   blue   ⓘ
SUCCESS  5s   green  ✓
WARN     8s   amber  △
ERROR    ∞    red    ✕   # sticky; user must dismiss
Reduce-motion (C24 a11y) drops the slide-in/out animation but keeps the timer sliver.
Inputs
Toast {
  severity   : INFO | SUCCESS | WARN | ERROR
  text       : RichString          # bold + 1 link allowed
  meta       : String?             # right-of-text small caps
  action     : { label, signal }?  # 1 inline action max
  ttl        : Float? (sec)        # default by severity
  group_key  : String?             # collapse duplicates
}
Outputs
signal action_clicked(toast_id)
signal dismissed(toast_id, reason) — reason ∈ {timeout, click, x, programmatic}
Group / dedupe
If a new toast has the same group_key as a visible one, the existing toast updates in place and its timer resets — instead of stacking duplicates. Used heavily by autosave + controller-state messages.
Edge cases
• Modal open (C25): toasts pause their timer and dim to 60% opacity until the modal closes. They don't disappear under it.
• Cutscene / portal transition (C20): queue is suppressed; on return, only ERROR toasts replay.
• HiDPI & safe area: anchor offsets respect SafeArea so handhelds don't clip the close button.
• Localization: width is fixed; long strings wrap up to 3 lines, then truncate with "…" and the toast becomes click-to-expand into the C26 history pane.
