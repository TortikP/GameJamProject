Spec — C25 Generic confirm
Purpose
Single Confirm dialog node, opened with a payload. Never hand-roll per-feature confirms — every "are you sure" goes through here so wording, focus rules, and audio cue stay consistent.
Anchor & sizing
Fixed-width 440 px, height auto. Centered on the screen above a 55% black scrim. Sits above C22/C23/C24 layers (z = modal). Only one C25 instance allowed at a time — second open requests are queued and shown after the first resolves.
Inputs
ConfirmRequest {
  severity     : Enum { INFO, WARN, DESTRUCTIVE }
  title        : String
  body         : String       # 1–3 sentences
  details      : String?      # optional lossy / metadata block
  buttons      : [Button]     # 2 or 3, ordered cancel→confirm
  default_btn  : ButtonId     # focus on open; never DESTRUCTIVE
  typed_token  : String?      # if set, confirm disabled until input matches
  tag          : String?      # right-aligned context chip ("unsaved · 2")
}
Outputs
signal resolved(button_id : String) — fires once, then modal closes. resolved("cancel") on Esc / scrim click / any non-positive close.
Severity → color
INFO        stripe = --ui-border-strong  primary = --ui-focus     (yellow)
WARN        stripe = --sem-debuff         primary = --ui-focus     (yellow)
DESTRUCTIVE stripe = --sem-damage         primary = .btn.danger    (red)
The stripe is the only chrome difference — body layout never changes between severities so muscle memory survives.
Focus & keyboard
• default_btn gets focus on open. For DESTRUCTIVE that's always Cancel.
• Esc resolves cancel.
• ↵ activates the focused button.
• Tab cycles within the modal only (focus trap).
• If typed_token is set, focus opens on the input. The Confirm button stays disabled until input == typed_token (case-sensitive). On match, focus moves to Confirm so ↵ fires immediately.
Edge cases
• Re-entry while open: incoming ConfirmRequest is queued, not stacked. Avoids the "wipe save → are you sure → are you sure" footgun.
• Keybind chip mid-capture: opening C25 cancels any pending C24 keybind capture (otherwise Esc would be eaten by the capture).
• Backdrop click: resolves cancel for INFO/WARN, but is ignored for DESTRUCTIVE — user must hit Cancel or Esc explicitly.
• Audio: open plays sfx_modal_open; DESTRUCTIVE additionally plays sfx_warn_pulse once.

Translation
All copy goes through tr(). The 4-char WIPE token is locale-aware — tr("CONFIRM_WIPE_TOKEN") — and the spec line uses tr("CONFIRM_TYPE_TO_CONFIRM", {token}). Don't bake "WIPE" into Godot script.
