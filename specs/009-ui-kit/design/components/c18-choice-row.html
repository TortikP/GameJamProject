C18 — Choice button row

The button strip used inside C16 dialogue. Numeric hotkey, label, optional requirement chip. Two layouts: horizontal (≤4 short choices) and vertical (long-form / >4 / requirement-bearing).

States

Horizontal · 3 short choices

Default for terse exchanges. Hotkeys 1–N. First choice focused on open.

Vertical · with requirements

When choices carry race / modifier / stat gates. Met = green chip, unmet = red + disabled.

Vertical · destructive

Choices that commit damage / consume resource get red hover border (no fill — affordance, not alarm).

Spec — C18 Choice button row
Purpose
Reusable choice strip — slotted into C16 dialogue and any narrative pause that needs branching.
Layout rule
• Horizontal when all labels < 24 chars and count ≤ 4.
• Vertical otherwise (default for any req-bearing choice).
Switch is automatic; no per-call toggle.
Inputs
choices: Array[Choice]
Choice {
  label:    String
  hotkey:   int        # 1-based
  req:      String?    # display chip; null = no chip
  met:      bool       # gates click + tints chip
  variant:  Enum { NORMAL, EXIT, DANGER }
}
Outputs
signal picked(index) — index in original array, regardless of layout.
Edge cases
• ≥10 choices: hotkeys go 1–9 then 0; further choices have no hotkey.
• All disabled: disable the row entirely; surface a hint above it (caller's responsibility).
• Focus on open: first met choice — never an unmet one.
Godot mapping

        Label hotkey

        Label req_chip               # hidden if null
