Spec — C24 Settings
Purpose
One panel for all tunables. Five tabs (Display / Audio / Gameplay / Controls / Accessibility). Apply / Revert / Defaults footer is constant.
Anchor & sizing
Centered modal, 720×auto (min-height 480). Opened on top of C22 or C23 — never replaces them.
Inputs
SettingsState  // single struct, nested per tab
KeybindMap     // action_id → { primary, alt }
Outputs
signal applied(diff), signal closed
Interaction rules
• Apply is enabled only while there are unsaved changes (footer counter).
• Closing with unsaved changes prompts a confirm (reuses C25 generic confirm modal).
• Keybind capture: click chip → "press a key…" state, swallow next keypress, validate against conflicts, surface conflict inline in footer meta.
Edge cases
• Conflicting bind: commit anyway but show red conflict meta; clicking the conflicting other action jumps focus to it.
• Defaults during run: warn that some changes (e.g. UI scale) may rearrange the in-progress turn UI; do not auto-revert.
• Controller present: Controls tab shows two columns (KB+M / Gamepad); same chip widget.
Godot mapping
SettingsPanel : CanvasLayer
  PanelContainer centered
    HBoxContainer
      VBoxContainer tabs (Button ×5, group)
      VBoxContainer pane                 # swapped per tab
    PanelContainer footer
      HBoxContainer (Label meta, Button ×3)
