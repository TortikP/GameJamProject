C8 — Cast-range / AoE overlay

Skill targeting overlay. Re-uses C7's reachable-tint primitive for range; adds a shape layer (single / cone / line / circle) painted in --sem-damage for offense, --sem-heal for support. Composes with C6 telegraphs (don't fight, don't hide).

States

Single target · range 3

Range tinted blue (in-reach hexes). Hovered enemy hex outlined damage-red. Hover any other in-range hex → no AoE preview.

Spec — C8 Cast-range / AoE overlay
Purpose
Show what the skill will hit before commit. Visual sibling of C7 movement preview — same hex primitive, different color semantics.
Anchor & sizing
World-space, drawn into the arena layer above tile sprites, below entities.
Inputs
caster_hex:    Hex
range:         int
shape:         Enum { SINGLE, CONE, LINE, CIRCLE }
shape_size:    int
intent:        Enum { OFFENSE, SUPPORT }   # picks color
target_hex:    Hex?
affects:       Array[Hex]                  # resolved AoE
Outputs
signal commit(target_hex), signal cancel
Composition rules
• C8 AoE fill never overlaps with C6 telegraph fill — when both present in the same hex, C6 wins z-order (player must always read enemy intents).
• Range tint is the same blue as C7; it's the shape fill that changes color by intent.
• If the AoE includes the caster (e.g. self-burst), self hex gets a faint green/red border + SELF label so it's not surprising.
Edge cases
• Range = 0 / self-only: skip range-tint pass; jump straight to AoE preview centered on caster.
• Line skills: commit on a direction-hex; AoE recomputes per frame.
• Mixed offense + support (rare): use damage red; surface heal targets via C4 HP preview only.
Godot mapping
CastPreview : Node2D                # mirror of MovementPreview, z = 6
  RangeLayer   : Node2D
  AoELayer     : Node2D            # color via shader uniform u_intent
  CenterRing   : Node2D            # animated dash; valid → focus, invalid → red
