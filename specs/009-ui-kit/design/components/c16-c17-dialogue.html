C17 Internal voice — proposals

Two options. Recommendation: option A (separate panel, top-anchored). Reason: voices have different narrative weight — second-person, no portrait, can interrupt mid-combat — so a distinct widget keeps the player's "I am being addressed by something inside me" cue legible.

A · Separate panel RECOMMENDED

Top-anchored, narrower (720). No portrait. Italic. Border tinted by absorbed entity's race.

— cultist · race: mortal "You felt his ribs give. That was not nothing. That was a sound she would have flinched from."
— hound · race: beast "More. More. The arena has corners, doesn't it?"

B · Mode of C16

Same 280px panel, no portrait, italic text, tinted border. Heavier visual presence; competes with NPC dialogue for the same screen real estate.

— cultist (in your head)
"You felt his ribs give. That was not nothing."
CLICK / SPACE — dismiss
Spec — C16 / C17
Purpose
C16: in-world dialogue with NPCs (portrait + name + text + choices). C17: internal absorbed-voice line (no portrait, italic, race-tinted).
Anchor & sizing
C16: bottom strip, full width minus margins, 280px tall.
C17 (recommended): top-anchored, 720 max-width, height grows with text (~80–140 px).
Inputs
// C16
speaker_name, portrait_tex, text, choices?, aux_tex?, auto_advance_sec
// C17
race, entity_name, text  # all italic, second-person convention
Outputs
signal choice_picked(index), signal advanced(), signal voice_dismissed()
Edge cases
• Mid-typewriter click → reveal full text without advancing.
• Mid-combat C17 fire — does NOT pause game; world keeps ticking. Voices auto-dismiss after 4s if no input.
• §12 trap: typewriter via RichTextLabel.visible_ratio, never visible_characters.
