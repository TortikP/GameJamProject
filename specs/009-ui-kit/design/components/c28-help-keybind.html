Spec — C28 Help / keybind
Purpose
A live, contextual cheatsheet. The bindings rendered here are read from the same InputMap C24 writes to — so a rebind in Settings instantly updates the help overlay too. No second copy of the truth.
Anchor & sizing
Compact: 360 wide, centered, ~240 tall.
Full sheet: 16 px inset from the viewport, 3-column grid, scrollable inside.
Inputs
InputMap                  // live, from project settings + C24 overrides
context : Enum {COMBAT, DIALOGUE, MODIFIER_PICK, MENU, SETTINGS}
input_kind : Enum {KB_M, GAMEPAD}   // last-used; auto-switches
Show / hide rules
• Hold (? or F1): show compact while held, hide on release. Min display 250 ms so a quick tap doesn't flash.
• Tap (same key): toggle full sheet on/off.
• Hidden during C25 confirm modal & C20 portal transition.
• Auto-hides if any binding is captured (C24 keybind capture).
Edge cases
• Unbound action: dashed chip with em-dash glyph. Clicking it (full sheet) opens C24 focused on that action's row.
• Conflicting bind: shows both keys side-by-side; the conflict-meta string from C24 footer reuses the same color.
• Mid-rebind: overlay can't open while C24 is in capture state — the capture would eat the help shortcut.
• Long action labels: the .desc sub-line wraps to 2 lines max, then truncates with "…".
Glyphs
KB+M: chip showing the literal key string ("Esc", "Space", "`", mouse buttons as "LMB"/"RMB"/"MMB").
Gamepad: face buttons as colored letter discs; shoulders/triggers as pill chips. Color is decorative — the letter is always present so colorblind users don't depend on hue.
Godot mapping
HelpOverlay : CanvasLayer  # singleton
  ColorRect scrim
  PanelContainer compact     # shown on hold
    VBoxContainer (Label header, GridContainer rows, Label hint)
  PanelContainer full        # shown on tap
    VBoxContainer
      HBoxContainer head
      HBoxContainer body (3 × VBox columns)
      HBoxContainer foot
