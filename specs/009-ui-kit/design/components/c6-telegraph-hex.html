Spec — C6 Telegraph hex
Purpose
Pillar 1 incarnate: every imminent effect is visible on the hex it will land on, with its number, before it resolves. Same widget telegraphs enemy intents and player cast previews (Pillar 2).
Anchor & sizing
World-space, parented to a hex via TileMap or Node2D at the hex's pixel center. Footprint = 1 hex (jam target ~80×92 at default zoom). Z-order: above terrain, below actors.
Inputs
hex_coord:  Vector2i
sources:    Array[{ source_actor_id: int,
                    skill_id: String,
                    primary_effect: enum,
                    amount: int,        # damage/heal numeric
                    duration: int,      # for status effects
                    label: String }]
visible:    bool
Display rules
primary_color = sem-{sources[0].primary_effect}
if all sources have type=damage:
  show single number = sum(amount)
  if len(sources) > 1: append "N sources" sub-label
elif effect is heal: show "+N"
else: show effect glyph + label like "stun 2t"
Glyph table: damage = numeric, heal = "+N", control = "✕", buff = "▲", debuff = "▼", move = "→→" or "←←", create = "◆".
Edge cases
• Mixed effect types on one hex (e.g. push + damage): show the damage number; secondary indicators are pip-icons in corner. Out of jam scope to design — flag for follow-up.
• Hex moves to fog-of-war: hide telegraph.
• Source actor dies before resolution: remove that source; if list empties, hide telegraph.
• Aggregation hover: when player hovers a multi-source hex, generic tooltip C15 shows breakdown (12 from Cultist, 7 from Hound).
Godot mapping
TelegraphHex : Node2D                  # positioned via hex_to_pixel()
  Polygon2D bg                          # 6-vertex hex, color = primary w/ alpha 0.22
  Line2D outline (optional)              # 1px hex outline, primary color
  Label num                              # 22pt mono bold, font_outline_size=2
                                            # anchor center; theme_override_color = primary
  Label src (optional)                    # 9pt mono, anchored below num
NO clip-path, NO gradients — Polygon2D handles the hex shape cheaply. Z-index: z_index = 1 (terrain=0, actors=10).
