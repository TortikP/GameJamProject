C7 — Movement preview path

World-space overlay shown while the player has a movement action queued and is hovering hexes. Three layers: (1) reachable hexes (blue tint, alpha by remaining cost), (2) chosen path (stronger blue with arrow segments), (3) destination ring (focus-yellow, animated).

States

Reachable area speed = 3

Player has 3 movement points. All hexes within 3 steps tinted blue. No path drawn until hover.

Spec — C7 Movement preview path
Purpose
Make the cost of a contemplated move legible before commit. Same primitive used for skill area-of-effect previews (C8) — share the renderer.
Anchor & sizing
World-space, drawn into the arena layer below entities, above tile sprites. Hex grid snapped via the existing HexMap helper.
Inputs
origin:    Hex
budget:    int
hovered:   Hex?
reachable: Array[{ hex, cost }]
path:      Array[Hex]
blocked_reason: String?
Visual rules
• Reachable hex alpha steps with remaining budget — closer = darker tint.
• Path arrow segments are drawn between hex centers, never along edges.
• Destination ring uses --ui-focus + animated dashes to differentiate from C6 telegraphs (which use --sem-damage).
Edge cases
• Hover own hex: no path, no ring.
• Path through difficult terrain: remaining-points HUD updates live; path may end short of cursor if budget exhausts.
• Out of reach: dest ring switches to red dashed.
• Race condition with intent telegraph (C6/C9): intent overlay drawn above movement preview.
Godot mapping
MovementPreview : Node2D            # child of arena, z = 5
  ReachableLayer : Node2D            # _draw() polys per reachable hex
  PathLayer      : Node2D            # _draw() polys + arrow segments
  DestRing       : Node2D            # custom-drawn animated dash
