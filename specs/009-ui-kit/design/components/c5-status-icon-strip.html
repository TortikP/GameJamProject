Spec — C5 Status icon strip
Purpose
At-a-glance readout of every status currently active on an actor, sortable, color-coded, with duration. Pillar 1: every timed flag that affects tactical decision is on screen.
Anchor & sizing
World-space, parented to actor. Centered horizontally above the HP bar, offset y = HP bar top − 6 px. Each icon 12×12 with 2px gap. Strip width = N × 14 − 2.
Inputs
statuses: Array[{ id: String,
                  kind: enum { buff, debuff, control, heal, damage },
                  glyph: String,        # 1 char, derived from id
                  duration: int,        # turns remaining
                  description: String   # for tooltip }]
max_visible: int = 6
Sort order
buffs → debuffs → control → others. Stable: insertion order within a bucket preserved.
Edge cases
• Empty list: render nothing, container collapses.
• > max_visible: show first 6, then "+N" chip with full list on tooltip hover.
• Duration = 1: apply expiring outline (yellow). Pillar 1 — last-tick warning.
• Permanent status (duration = -1): show "∞" glyph in dur field, no outline.
• Status updates mid-frame: HP bar position tracks the strip width — re-anchor on change.
Godot mapping
StatusStrip : HBoxContainer            # separation = 2; horizontal = SHRINK_CENTER
  StatusIcon ×N : Control               # custom_minimum_size = (12, 12)
    ColorRect bg                          # color = sem-{kind}
    Label glyph                           # 9pt mono bold, color black
    Label dur                             # 8pt mono, anchor bottom_center y=+8
    Control outline                       # StyleBoxFlat 1px black, +1px focus when expiring
                                            # tooltip_text or _make_custom_tooltip()
  OverflowChip (Label "+N")              # visible only if statuses.size() > 6
Connect to actor's statuses_changed signal; rebuild children.
