Spec — C27 Combat log
Purpose
Forensic record of combat. Players use it to ask "why did I just die" — so the log must surface crits, dodges, armor reduction, and tick damage clearly. Every numeric in the log matches the C11 float that fired at the same instant.
Anchor & sizing
Collapsed pill: bottom-left of viewport, 360 wide, height 26. Expanded panel: 460×480, anchored bottom-left as well; the pill is the panel's footer when expanded so spatial origin stays consistent. Tab toggles between states.
Inputs
LogEvent {
  ts        : Float          # seconds since wave start
  turn      : Int
  category  : Enum { DMG, HEAL, CTRL, BUFF, DEBUFF, MOVE, MISC }
  actor_id  : ActorId
  target_id : ActorId?
  verb      : I18nKey        # tr("LOG.CLEAVE_HIT")
  numbers   : { value, kind } []   # kind ∈ {dmg, heal, crit, ...}
  flags     : { crit, dodge, immune, armor_reduced } bitset
  source    : SkillId | StatusId | "tile"
}
Outputs
signal row_clicked(actor_id) → opens C12 actor inspector for that actor at that timestamp.
signal copy_requested(scope) → flat plain-text log to clipboard for bug reports.
Filtering & scope
• Scope: Turn (current) / Wave / Run. Run scope is paginated 100 events per page.
• Filters: 7 category chips, persisted per profile. Off chips hide rows but never collapse turn separators (so timing context survives).
• Crit, dodge, immune are flags, not categories — they always show on whichever line they belong to.
Edge cases
• Identical event burst: 3+ same-source rows in < 0.4 s collapse into one with "×N" and a sum (e.g. tick bleed ×3 · −9). Click to expand inline.
• Reduce-motion: new lines fade-in instead of slide-in.
• Pause / portal: log is inert during C20 transitions; no fake "wave 4 cleared" rows.
• Color-blind palette (C24): rows tint via the corresponding remapped semantic tokens; no hard-coded hex.
Persistence
Events are kept per run. Run end → C21 summary can paginate the full log. After C21 close, the buffer is dropped.
