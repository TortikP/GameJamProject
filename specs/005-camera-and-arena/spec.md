# 005-camera-and-arena — spec

**Owner:** Andrey (предложить Alexey/Egor — короткая presentation-фича)
**Status:** draft (без имплемента)

## Цель

Добавить в Godmode-сцену управляемую камеру с зумом колесом мыши и расширить базовую гекс-арену, чтобы появилось пространство для тестирования speed/AI/манекенов. Сейчас арена 8×5 — для проверки `speed >= 2` и нескольких манекенов одновременно тесно.

Это чисто presentation-слой — не трогает `core/`.

## Acceptance criteria

- В `scenes/dev/godmode.tscn` присутствует `Camera2D` (`current = true`, `enabled = true`), child от `Godmode` Node2D.
- Стартовая позиция камеры — центр игрока (на тайле спавна).
- **Колесо вверх** = zoom in, **колесо вниз** = zoom out. Шаг и пределы — из `config/game_speed.cfg`, секция `[godmode]`: `zoom_step`, `zoom_min`, `zoom_max`.
- Зум — плавный (interpolation через `Tween`, длительность из `[godmode] zoom_lerp_duration`). Не дёрганый «прыжок» на каждое тиканье колеса.
- **Зум к курсору, не к центру:** точка под курсором мыши до зума остаётся под курсором после зума. Проверка: подвести курсор к углу карты, крутить колесо — угол не уезжает из-под курсора.
- Зум за `zoom_min`/`zoom_max` не выходит (clamp). При попытке выйти — visual no-op, без логов-спама.
- Карта Godmode расширена: `GRID_W = 14`, `GRID_H = 9` (раньше 8×5). Игрок спавнится в `(7, 4)` (центр).
- `HexCursor` (presentation) корректно работает при любом зуме — пересчёт через `coord_under_mouse()` уже учитывает viewport-трансформацию.
- F5 hot-reload `game_speed.cfg` подхватывает новые `zoom_*` ключи без перезапуска (поскольку они читаются на каждое колесо, не кэшируются).

## Out of scope (этот PR)

- **Pan камеры** (drag MMB / WASD / стрелки) — отдельная фича. Сейчас если игрок ушёл далеко — камера за ним не следует.
- **Following camera** (камера автоследования за игроком после move) — отдельная фича; обсудить нужна ли вообще.
- **Загрузка размера сетки из JSON / config** — пока константы в `godmode_controller.gd`. Если позже понадобится конфиг — отдельная фича.
- Применение Camera2D в **других** сценах (`arena_demo.tscn`, будущая боевая сцена) — отдельный PR. Здесь правим только Godmode.
- Camera shake / cinematic эффекты / smooth follow — поздняя polish-фича.
- Зум через клавиши (+/−) — можно добавить во второй итерации, не блокирует.

## Acceptance verification

Запуск `scenes/main.tscn` → кнопка "Godmode":
1. Сцена открывается, виден игрок в центре карты 14×9, камера сцентрирована.
2. Колесо вверх 5 раз → плавный zoom in, точка под курсором стабильна.
3. Колесо вверх ещё 50 раз → дальше `zoom_max` не уходит.
4. Колесо вниз → плавный zoom out, аналогично clamp на `zoom_min`.
5. RMB-движение / LMB-каст / F1-спавн / QWER-выбор слота — работают на любом уровне зума без визуальных артефактов.
6. F5 → правка `zoom_step` в `game_speed.cfg` → следующее тиканье колеса использует новый шаг.

## Зависимости

- 002-hex-grid (HexGrid с `coord_under_mouse()`).
- 004-godmode-base (Godmode-сцена, GameSpeed config с секцией `[godmode]`).

## Что считается breaking

- Изменение `GRID_W/GRID_H` — только в Godmode-сцене, на core API не влияет. **Не breaking.**
- Добавление ключей в `[godmode]` секцию `game_speed.cfg` — additive. **Не breaking.**
- Camera2D как новый child Godmode-сцены — additive. Если кто-то полагался на отсутствие камеры (нет таких) — поломал бы. Не наш случай.

## Заметки

Расширение карты до 14×9 — оценочный размер для тестов: видно 4 манекена + игрок + есть куда отступать при `speed=2`. Если на ревью покажется тесно/просторно — поправить константы тривиально.
