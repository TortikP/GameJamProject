## v3 — итерация по скриншоту #2

- [x] T060 [P0] **Яркое пятно в центре было кругом, должно повторять 16:9-кадр.**
      В v2 я aspect-коррекцией сделал виньетку равномерной в физических пикселях
      (физический круг → визуально вертикальный овал в UV). Это противоположно
      нужному. Откатил: использую `length(from_center)` прямо в UV. На 16:9-экране
      UV-окружность рендерится как горизонтальный овал, повторяющий форму экрана
      (0.5 UV = 640 px по горизонтали vs 360 px по вертикали → бoльшой простор по бокам,
      зона ярко-горящего фосфора вытягивается под кадр). Заодно подобрал пороги
      smoothstep'а — 0.25..0.85 от length вместо 0.45..0.95 от length·1.6.

## v5 — оптимизация + чистка краёв

Скрин показал лаги и артефакты на краях (бахрома хром-аберрации + блюр-смазывание
на мелком тексте у бортов кинескопа).

- [x] T080 [P0] **Хром-аберрация: 3 → 2 текстурных сэмпла.** Берём один сэмпл с
      `+chroma_offset`, один с `-chroma_offset`. R = sa.r, B = sb.b, G = (sa.g + sb.g)*0.5.
      Визуально неотличимо от полноценного 3-сэмпл подхода, экономия 1 текстурного
      чтения на фрагмент.
- [x] T081 [P0] **Bloom: 4 → 2 сэмпла + gate.** Диагональные семплы вместо креста.
      Главное: всё это теперь под `if (bloom_strength > 0.0)` — выкручивание в 0 в
      инспекторе реально выключает работу, а не «обнуляет результат после семплинга».
- [~] T082 [P0] **Early-exit за безелем.** Был `if (bz >= 1.0) { COLOR = black; return; }`
      сразу после `bezel_mask`. **Откатил** — на машине Андрея шейдер не компилился
      (чёрный экран как fallback к ColorRect.color). Подозреваю что Godot 4.6
      gl_compatibility GLSL-конвертер плохо переваривает `return;` из `void fragment()`.
      Не критично — основной перф-выигрыш дают T080/T081/T083, бекл-работа в безеле
      (~5% фрагментов) — мелочь.
- [x] T083 [P0] **Хром затухает к краю.** `chroma_offset *= screen_mask` — у самой
      границы безеля смещение → 0 → сэмплы не уезжают за `clamp_to_edge` →
      пропадают «протяжки» крайнего пикселя на бортах.
- [x] T084 [P0] **Bloom радиус 3 → 2 px.** Тайтнее свет, лучше для пиксель-арта,
      меньше cache-misses.

Итого по текстурным сэмплам внутри экрана: 7 → 4. По безелю: → 0.
−40% по bandwidth для основной зоны и почти полное обнуление работы для рамки.

## v6 — clamp-to-edge stripe на левом борту

После v5 на скрине осталась вертикальная жёлтая полоса по левому краю — это
clamp-to-edge артефакт: barrel-curvature на левом борту задвигает `cuv` слегка
за 0 (`cuv.x ≈ -0.01`), сэмплер с `repeat_disable` подхватывает крайнюю колонку
и тянет её вертикально. Для хром-аберрации в v5 я это закрыл (`chroma_offset *= screen_mask`),
но bloom-сэмплы остались на фиксированном радиусе, плюс базовая `cuv` сама по себе
уезжала в OOB до того как bezel_mask успевал дотемнить.

- [x] T090 [P0] **Bloom радиус ×= screen_mask.** Тот же приём что и для chroma в v5 —
      у бортов диагональные сэмплы стягиваются к одной точке, не уезжают за край.
- [x] T091 [P0] **`cuv_safe`: clamp перед сэмплингом.** Все texture-вызовы теперь идут
      от `clamp(cuv, sample_margin, 1 - sample_margin)`, где margin ≈ 4 px. Если cuv
      ушёл в OOB, мы сэмплим с инсета (реальный игровой контент), а не реплицируем
      крайнюю колонку. Видимый сдвиг в 4 px на самом борту — незаметен.
- [x] T092 [P0] **Bezel inset = 0.005.** Bezel_mask теперь начинает гаснуть не на UV=0,
      а на UV=0.005 — стрипа-зона визуально перекрывается чёрной рамкой. Видимая
      площадь экрана теряет 0.4% по горизонтали — глазом не видно.

## v7 — pre-shrink в barrel(), фикс корня проблемы

Все предыдущие меры с краями (`cuv_safe`, `screen_mask`-fade на офсетах,
инвертированный `bezel_mask` из 22e3da9) — это компенсация **снаружи**: пытались
маскировать ситуацию когда `cuv` уехал за [0,1]. Корень проблемы: барель сам
по себе расширяет UV — на углах при curvature=0.08 даёт scale=1.04, и `cuv`
вылезает в `(1.02, 1.02)`. Любая текстурная выборка там автоматически клампится.

- [x] T100 [P0] **Pre-shrink в `barrel()`.** До применения квадратичного расширения
      делим базовый `(uv-0.5)` на `1 + curvature*0.5`. На углах после расширения
      `cuv` гарантированно в `[~0.001, ~0.999]` — сэмплы legitimно внутри текстуры,
      clamp-to-edge физически невозможен.
- [x] T101 [P0] Видимая картинка сжимается на ~2% по каждой оси. Незаметно на
      фоне закругления и безеля. Альтернатива — SubViewport-rebuild — стоит
      полудня переделок и не нужна.

## v7 — финал краёв: hard 1 при любом OOB

В v6 `bezel_mask` всё ещё разрешал 30–90% видимости в OOB-зоне, а сэмплы там
неизбежно идут из `cuv_safe` (одна и та же колонка) → стрит-полоса жила.

- [x] T100 [P0] **Перевёрнутая логика `bezel_mask`.** Теперь:
      `dist_inside = max(-oob, 0)` — расстояние от ближайшего края (0 если OOB).
      `bz = 1 - smoothstep(0, bezel_softness, dist_inside)`.
      Любой fragment с `cuv` ⩾ за [0,1] получает `bz = 1` (полный безель), screen_mask = 0.
      Никакая `cuv_safe`-выборка теперь не имеет значения — её результат
      умножается на 0. Артефакт убит на корню.
- [x] T101 [P0] **bezel_softness 0.04 → 0.025**, чтоб видимое тёмное кольцо по
      периметру осталось примерно той же толщины, что было визуально в v6
      (раньше 0.04 — это была ширина OOB-зоны fade'а, сейчас — ширина внутреннего
      fade'а). На 1280-px viewport ≈ 32 px на сторону. Читается как настоящее
      стекло кинескопа, не как баг.
