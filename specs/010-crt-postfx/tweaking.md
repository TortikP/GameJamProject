# 010-crt-postfx — tweaking guide

Где открыть и что крутить.

## Где найти ручки

1. Открой `scenes/presentation/crt_post_fx.tscn` в Godot.
2. В дереве: `CrtPostFx → Screen` (ColorRect).
3. В инспекторе раскрыть `Material → ShaderMaterial → Shader Parameters`.

Все параметры применяются вживую при редактировании сцены — Godot перезагружает
шейдер автоматически. Сохранить сцену чтобы значения остались между запусками.
F6 в игре — выкл/вкл всего эффекта целиком.

## Какие крутилки и зачем

### Геометрия экрана

- **`curvature`** (0.0..0.5, дефолт 0.08) — насколько выпуклый кинескоп. 0 = плоский,
  0.15 = старый Sony, 0.25+ = карикатурно круглый. Внутренний `barrel()` сжимает
  UV до применения, чтобы изображение всегда оставалось в пределах текстуры —
  можно гнуть сильно, без артефактов на краях.
- **`bezel_softness`** (0.0..0.2, дефолт 0.025) — толщина мягкой чёрной рамки
  вокруг изображения. 0.025 ≈ 32 px на 1280-широком вьюпорте. Если хочется
  чтобы кадр заполнял почти весь экран — крути в 0.005. Если хочется широкую
  чёрную рамку — 0.05+.

### Сканлайны

- **`scanline_pitch_px`** (1.5..8.0, дефолт 3.0) — расстояние между тёмными
  линиями в **физических пикселях окна**. 2 = частые линии для ретро-консоли,
  4–5 = более редкие для пиксель-арт-эстетики. Привязка к FRAGCOORD.y, а не
  к UV — никакого муара, всегда ровные линии независимо от размера окна.
- **`scanline_strength`** (0.0..1.0, дефолт 0.30) — глубина потемнения. 0 =
  выкл, 0.5+ = очень читабельные, тёмные линии.

### Aperture (RGB-маска)

- **`aperture_strength`** (0.0..1.0, дефолт 0.15) — насколько видна вертикальная
  RGB-полосочка. 0 = нет, 0.30+ = триплеты «прыгают в глаза», ощущение крупного
  фосфора. Триплеты `mod(FRAGCOORD.x, 3)` — то есть 1 триплет на 3 физических пикселя.

### Хроматическая аберрация

- **`chroma_strength`** (0.0..6.0, дефолт 1.6) — пиксельный сдвиг между R и B
  каналами. 0 = выкл, 3+ = сильный rainbow-эффект на тексте. Сила пропорциональна
  расстоянию от центра — в центре всегда чисто. Сэмплы автоматически фейдятся
  у края (через `screen_mask`), артефактов на бортах быть не должно.

### Тёплый тон (главный «лампово»-параметр)

- **`warm_strength`** (0.0..1.0, дефолт 0.28) — сила тёплого grade'а. 0 = чисто
  RGB, 0.5+ = как у бабушки в шкафу.
- **`warm_tint`** (color, дефолт rgb(1.18, 1.04, 0.78)) — какой именно тёплый.
  Сдвинь green вниз → больше оранжа. Подними blue → ближе к нейтралу.
- **`phosphor_glow`** (color, дефолт rgb(0.030, 0.018, 0.008)) — постоянный
  янтарный свет в чёрных областях. Это «фосфор не отдыхает». Подкрути в 2x для
  сильнее «горящего» ощущения. В 0 — чёрный будет настоящим чёрным (потеряется
  лампово-эффект).

### Bloom (свечение ярких пикселей)

- **`bloom_strength`** (0.0..1.5, дефолт 0.55) — сила свечения. 0 = выкл, 1+ =
  яркие места «текут» по экрану как на старом мониторе. **Установка в 0
  отключает работу шейдера в этой ветке** — реальная экономия perf'а.
- **`bloom_threshold`** (0.0..1.0, дефолт 0.45) — порог яркости, выше которого
  пиксели начинают цвести. Меньше = больше пикселей цветут.

### Vignette (затемнение углов)

- **`vignette_strength`** (0.0..2.0, дефолт 0.42) — сила затемнения углов. UV-круг,
  на 16:9 рендерится как горизонтальный овал — повторяет форму кадра.
  0 = выкл, 1.0+ = «тоннель», ярко только в центре.

### Tone shaping

- **`boost`** (0.5..2.5, дефолт 1.5) — компенсация яркости. Стек умножений
  (scan × aperture × vignette × pow) даёт −40% яркости — boost восстанавливает.
  Если картинка кажется выгоревшей — режь до 1.2. Если мутная — поднимай до 1.7.
- **`contrast`** (0.5..2.0, дефолт 1.12) — pivot-at-0.5 контраст. 1.0 = нет
  изменений. 1.25 = ощутимый pop. <1 = мягче, как ламповый монитор начала 2000-х.

### Гул сети (50 Hz hum)

- **`hum_strength`** (0.0..0.2, дефолт 0.04) — амплитуда модуляции яркости.
  0 = выкл (бесплатно, без работы), 0.04 = чуть заметные полосы яркости,
  0.10+ = заметная пульсация (может бесить).
- **`hum_speed`** (0.30 дефолт) — скорость прокрутки полос. Меньше = медленнее.
- **`hum_bands`** (2.0 дефолт) — сколько синусоидальных полос видно одновременно.
  1 = одна большая волна по экрану, 3+ = несколько частых полос.

## Пресеты-ориентиры

**«Subtle, для скриншотов»**: curvature 0.04, scanline_strength 0.18,
aperture 0.08, chroma 0.6, warm 0.18, bloom 0.30, vignette 0.30, boost 1.2,
contrast 1.05, hum 0.0.

**«Текущий дефолт»** (читать выше): тёплый, заметный, без перебора.

**«Old TV Trinitron»**: curvature 0.12, scanline_pitch 4.0, aperture 0.30,
warm_strength 0.40, warm_tint (1.22, 1.05, 0.72), phosphor_glow ×2,
bloom 0.7, vignette 0.55, hum 0.06.

**«Совсем кринж-ретро»**: curvature 0.20, aperture 0.45, chroma 3.0,
scanline_strength 0.5, hum 0.10. Не для прода, для стрима/мема.

## Перф

Если игра тормозит на конкретной машине:
1. `bloom_strength = 0` — самая дешёвая экономия (отключает 2 текстурных сэмпла на фрагмент).
2. `aperture_strength = 0` — это арифметика, perf не растёт, но визуально проще.
3. F6 — выкл всего эффекта, мгновенный возврат к нативному рендеру.
