# 010-crt-postfx — spec

**Owner:** Andrey
**Status:** in progress

## Цель

Полноэкранный пост-эффект «выпуклый ЭЛТ-телевизор» поверх любой сцены игры.
Тёплый ламповый вайб: barrel-кривизна, сканлайны, aperture-mask, хром-аберрация,
vignette, чёрный bezel-край. Атмосферный слой — добавляет ретро-телевизионную текстуру
ко всему происходящему.

## Acceptance criteria

- В любой сцене (`main_menu`, `arena`, `meta`, `dialogue`) виден ЭЛТ-эффект:
  кривизна экрана, сканлайны, виньетка, тёплый тон, чёрные углы за изогнутой
  «рамкой кинескопа».
- Эффект включается/выключается на лету по F6 без артефактов и без перезагрузки сцены.
- Состояние (вкл/выкл) и сила параметров доступны через api `CrtPostFx`
  (autoload-нода).
- Все косметические параметры (curvature, scanline_strength, aperture_strength,
  vignette_strength, chroma_strength, warm_strength, bloom_strength) экспортированы
  как `uniform` в шейдере и видны в инспекторе ColorRect — Андрей крутит на лету.
- При выкл — нулевые накладные расходы (CanvasLayer.visible = false, шейдер не
  выполняется).
- При вкл — на 1280×720 в окне 1080p стабильные 60 fps на интегрированной графике
  (визуально, без замеров — если просядет ниже 60 на нормальном железе, режем
  bloom-таблицу).
- Не ломает клики мыши: UI-узлы под ColorRect получают input как обычно
  (mouse_filter = MOUSE_FILTER_IGNORE на пост-rect-е).
- Не ломает уже существующий рендер: GL Compatibility, `stretch/mode="viewport"`
  остаются как есть, ни одну существующую сцену не пересобираем.

## Out of scope

- SubViewport-пайплайн с попиксельной привязкой сканлайнов к игровым пикселям
  (минимум полдня переделки, для джема не окупается; см. plan.md §«Альтернатива»).
- Анимация включения/выключения телевизора (vintage CRT power-on/off с белой
  точкой посередине). Можно добавить в `011-crt-power` если останется время в
  субботу — это уже polish-polish.
- Toggle в Settings-panel — отдельная задача `P1` в `tasks.md`. На P0 хватает F6.
- Звуковой компонент (электростатический гул, щелчок при включении). Если
  Андрей решит — отдельная задача под AudioDirector, не в этом спеке.
- Переходы (тру-CRT с roll'ом картинки при смене сцены). Полировка следующего уровня.
- Поддержка мобильного rendering_method.mobile — бэкенд тот же gl_compatibility,
  должно работать «бесплатно», но не верифицируется в рамках этой задачи.
