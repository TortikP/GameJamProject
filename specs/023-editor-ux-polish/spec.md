# 023-editor-ux-polish — spec

**Owner:** Andrey.
**Status:** ACTIVE — feature spec. P1 уже частично сделан в той же ветке (`andrey/023-editor-ux-polish`, draggable panels + layout fix).

## Цель (одно предложение)

Снять трение с map editor (020) до плейтест-уровня: убрать раздражающие модалки, добавить недостающие базовые жесты редактора (drag-paint, undo, hotkeys, eyedropper) и привести палитру объектов к виду, в котором по силуэту видно, что положил.

## Что входит

Разделено на P1/P2/P3 по импакту. P3 берём только если P1+P2 закрыты с запасом.

### P1 — без этого редактор рабочий, но больно

- **Drag-paint LMB.** Зажатие ЛКМ → рисует пол / ставит объект на каждом гексе под курсором. Erase-режим тоже drag-able. Уникальные сущности (player spawner) при drag не дублируются.
- **Silent reject занятого тайла.** Модалка «Тайл занят» удалена. Промах на занятый = ничего не происходит, опционально мягкий toast 1× на серию (debounce, чтобы при drag не сыпался спам).
- **Undo/Redo (Ctrl+Z / Ctrl+Y).** Snapshot-стек до 50 шагов. Каждое мутирующее действие пушит snapshot целого `LevelData`. Redo-стек чистится при новом действии. Drag-paint = одна транзакция (один snapshot на серию, не на каждый гекс).
- **Ctrl+S → save без модалки.**
- **Dirty-asterisk в имени уровня.** `* Untitled` пока есть несохранённые изменения; на Save → `Untitled`.

### P2 — заметно ускоряет

- **Eyedropper (Alt+ЛКМ).** Под курсором пол → выбран соответствующий tile в палитре. Под курсором object/spawner → выбран в Objects-tab. Уровень не модифицирует.
- **Quick palette select (1-9).** Цифровые клавиши переключают на N-ю кнопку текущей видимой палитры (Floor или Objects, по контексту).
- **Hotkey overlay (H).** Тоггл полупрозрачной шпаргалки поверх экрана со списком всех шорткатов.
- **Тематический фон панелей.** Текущий `make_panel_stylebox` плоский серый. Тонкий accent-border + лёгкая внутренняя тень. Меняется один раз в `UiTheme`, подхватывается всеми панелями.
- **Силуэты объектов вместо hash-квадратов.** Каждый `TileObject` рисуется как простая монохромная фигура (круг / треугольник / ромб / прямоугольник) с цветом по тегам. Без участия Кати. Реализация — `_draw()` в `ObjectsOverlay`, маппинг `object_id` → `shape_kind`.
- **Новые объекты (контент)**: `tree` (breakable wood, hp 3, on_destroy nothing), `bush` (small walkable, lightweight cover или slow), `crystal` (large breakable, лёгкая аура). Через существующую схему `data/tile_objects/*.json`, без кода.
- **Sample enemies**: 2-3 stub `data/enemies/*.json` (skeleton, slime, archer) — проверка, что auto-discovery спаунеров в редакторе действительно работает (сейчас всего один манекен в данных, нечем проверить).

### P3 — берём, если осталось время

- **Drag-existing objects** (P3 stretch из 020, T019).
- **Box-select** для batch-операций.
- **Connectivity warning** — подсветка изолированных островов перед save.
- **Map preview thumbnail в Load FileDialog.**
- **Дополнительные tile types** (grass / sand / stone) — нужны ассеты Кати или временный programmatic tileset. Пока пропускаем.

## Acceptance criteria (P1+P2)

1. Зажал ЛКМ, провёл по 10 гексам — все 10 закрашены/заполнены за один проход. Один `Ctrl+Z` отменяет всю серию, не один гекс.
2. Тыкнул на занятый гекс — модалка не появляется, состояние уровня не меняется.
3. `Ctrl+S` сохраняет без всплывающих окон.
4. После любой мутации в имени уровня появляется `*`. После Save — пропадает.
5. `Alt+click` на гекс с tree → в палитре Objects активна кнопка `[L] tree` (или соответствующая).
6. Нажатие `H` показывает оверлей со списком всех шорткатов; повторное `H` скрывает.
7. На карту положены все 6 существующих объектов + 3 новых из P2 — каждый узнаётся по силуэту, не как hash-квадраты.
8. Создан второй файл `data/enemies/skeleton.json` → в редакторе появляется `Spawn: skeleton` без правок кода (после рестарта редактора, hot-reload не требуется).

## Out-of-scope (явно не делаем в этой спеке)

- Hot-reload `data/enemies/` без рестарта редактора.
- Сохранение позиций draggable-панелей между сессиями.
- Resize / collapse панелей.
- Multi-tile brush (3×3 fill, etc).
- Mirror painting / random scatter / fill bucket.
- Per-actor overrides на спаунере (HP, скиллы конкретного манекена).
- Animated objects.

## Зависимости

- 022 (hex-shape) — стабилен, уже в staging. Силуэты объектов рассчитывают центр гекса через `tile_map_layer.map_to_local`, форма гекса не блокирует.
- 020 (map editor) — основа, расширяем без слома публичного API палитр.
- 018/019 (tile-objects + resolver) — содержат регистр объектов, никаких новых полей в схеме.

## Размер

P1 ~3-4 часа, P2 ~4-5 часов, P3 — по остатку. Всё умещается в один-два захода работы Андрея перед плейтестом.
